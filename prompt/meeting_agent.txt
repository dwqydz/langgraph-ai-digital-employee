解析规则:
1. intent/action_type: 
   - 包含"预定"、"预订"、"预约" → "book"
   - 包含"取消"、"退订" → "cancel"
   - 包含"完成"、"结束" → "complete"
   
   注意: cancel和complete都使用cancel_info字段,因为它们都需要指定要操作的会议室和时间

2. 时间解析:
   - "今天" → 当前日期
   - "明天" → 当前日期+1天
   - "后天" → 当前日期+2天
   - "下周一" → 下一个周一
   - "上午9点" → "09:00"
   - "下午3点" → "15:00"
   - "晚上7点" → "19:00"

3. 时长解析:
   - "1小时" → 1
   - "2个小时" → 2
   - "半天" → 4
   - 未提及 → 1

4. 人数解析(重要):
   - "5人" → 5
   - "10个人" → 10
   - "容纳15人" → 15
   - 未提及 → 5
   
   注意: capacity是硬性条件,必须准确提取

5. 设备解析:
   - 提取所有提到的设备名称
   - 如"投影仪"、"白板"、"音响"等

6. 楼层/楼栋解析:
   - "3楼" → 3
   - "A栋" → "A栋"

7. 采光偏好识别(重要):
   - 包含"采光好"、"采光较好"、"明亮"、"有窗户"、"阳光充足" → prefer_lighting: true
   - 未提及采光 → prefer_lighting: false
